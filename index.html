What is a Web Component?

In a nutshell Web Components, also called custom components, are a modern standard that allows developers to encapsulate and reuse unique functionality in web apps. Instead of having to duplicate large amounts of HTML, nest various elements, or create multiple JavaScript functions and event listeners, developers can now rely on Web Components.

A more technical explication: Web Components use serval modern web technologies (Custom elements, Shadow DOM, and HTML Templates) to allow developers "to [easily] create reusable custom elements – with their functionality encapsulated away from the rest of your code – and [utilized] in [any] web [application]." (MDN, Web Components)

Component vs. Web Component

You have probably heard the term component being used before. In web design it usually means reusable code (the component) that encapsulates a particular feature or functionality. This component can then be added to any web page to provide the feature or functionality to the users. Every major web development framework (some examples pictured below) either offers the ability to build with or create components that adhere to the frameworks specifications.

None of these frameworks use actual Web Components – they are actually one of the biggest reasons the Web Component standard was created – but they all can implement (use) Web Components; some more easily than others. A Web Component, or custom component, is a native web standard and does not rely on a framework, library, transpiling, or compiling to work.

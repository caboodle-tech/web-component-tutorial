Scenario

Your building an application that requires gathering large amounts of information from users. It has been determined that adding tool tips to some of the more difficult questions would aid users in completing the forms faster. Your team has decided they want the tool tips to look identical and sit inline with the questions to improve the users experience.

Code Comparison

If you created these tool tips manually each one would have HTML that looks like the following example. You would then need to add CSS rules to hide the summary marker and inline the details element: 

<details>
    <summary>
        <svg xmlns="http://www.w3.org/2000/svg" width="24" height="24" viewBox="0 0 24 24"><path d="M12 2c5.514 0 10 4.486 10 10s-4.486 10-10 10-10-4.486-10-10 4.486-10 10-10zm0-2c-6.627 0-12 5.373-12 12s5.373 12 12 12 12-5.373 12-12-5.373-12-12-12zm-.001 5.75c.69 0 1.251.56 1.251 1.25s-.561 1.25-1.251 1.25-1.249-.56-1.249-1.25.559-1.25 1.249-1.25zm2.001 12.25h-4v-1c.484-.179 1-.201 1-.735v-4.467c0-.534-.516-.618-1-.797v-1h3v6.265c0 .535.517.558 1 .735v.999z"/></svg>
    </summary>
    <p>
        Tool tip message here...
    </p>
</details>

Using our custom tool-tip element all that has to be added to the page is the following HTML. The icon is added automatically for us as well as the styles: 

<details is="tool-tip">
    Tool tip message here...
</details>

Points to Ponder

  • This is an incomplete example. If you demo this page on a mobile device you will see that our tooltip overflows the page and becomes illegible. This can be fixed by adding an event listener to our custom element that will position the tool tip appropriately when its opened.

  • Because we used the details element here we can not take advantage of a shadow DOM; only select elements support adding a shadow DOM. The drawback is we need to add a lot of inline styles. The benefit is our element maintains user accessibility without having to manage ARIA because this is a semantic element already.

  • We could improve the styling dilemma from the last point by extending an HTML element that supports adding a shadow DOM. If we make this change, since we are no longer extending a semantic element, we will have to make sure we include ARIA support ourselves.

Points to Ponder

  • This is an incomplete example. If you demo this page on a mobile device you will see that our tooltip overflows the page and becomes illegible. This can be fixed by adding an event listener to our custom element that will position the tool tip appropriately when its opened.

  • We improved the styling dilemma and simplified our custom elements class compared to the built-in demo by using the shadow DOM.

  • Because we made a truly custom element not extended from a built-in semantic element, we had to manage accessibility support ourselves. We used the semantic button element with role="tooltip" and a custom event listener to toggle the tool tip open and closed. We also had to toggle the aria-hidden attribute on the tool tip p element.

    This example by no means accounts for all edge cases of user accessibility, only the bare minimum has been done to create a functional demo. For more information please explore the ARIA documentation.

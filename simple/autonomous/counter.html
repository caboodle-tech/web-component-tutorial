Scenario

Your part of a team that is helping build software for carnivals and fairs. You have been tasked with creating a reusable counter component that can be used by ride operators and gate attendants to quickly count how many people rode a ride or entered a particular gate.

Demo

Since an autonomous web component does not extend an existing element, there is no comparison between a built-in HTML element and our custom element in this demo. Here is our custom keep-count element in action:

Code

This is all the HTML needed to create this custom element:

<!-- We just have to reference our custom component --> <keep-count></keep-count>

After our custom element is render this is the HTML that is automatically put in place:

<keep-count> #shadow-root (open) <style> button { display: inline-flex; align-items: center; justify-content: center; font-weight: bold; font-size: 2rem; width: 2.25rem; height: 2.25rem; margin-right: 15px; cursor: pointer; } div { display: inline-block; font-weight: bold; font-size: 2rem; } </style> <div> <button>-</button> <button>+</button> <div>0</div> </div> </keep-count>

Points to Ponder

  • This is an incomplete example. We could add additional styles, features, and connections to the counter. A simple feature we could add is saving the current count in local storage and restoring it on page reloads. An advanced example is we could have counters sync with an API to auto report their numbers.

  • User accessibility needs to be addressed for this component. We used button elements to increase or decrease the counter so there is partial support, but we are missing ARIA roles to help visually impaired users understand what those buttons actually do.

  • We improved styling and simplified our custom element by using the shadow DOM.
